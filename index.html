Scalable Real2Sim:
Physics-Aware Asset Generation
Via Robotic Pick-and-Place Setups

Nicholas Pfaff1,   Evelyn Fu1,   Jeremy Binagia2,   Phillip Isola1,   Russ Tedrake1
1Massachusetts Institute of Technology, 2Amazon Robotics
arXiv Video Code Benchmark Dataset

Abstract

Accurately simulating object dynamics from real-world perception inputs shows great promise for applications in digital twins and robotic manipulation but typically requires labor-intensive measurements and domain expertise. We present a fully automated Real2Sim pipeline that generates simulation-ready assets for real-world objects through robotic interaction. Using only a robotâ€™s joint torque sensors and an external camera, the pipeline identifies visual geometry, collision geometry, and physical properties such as inertial parameters. Our approach introduces a general method for extracting high-quality, object-centric meshes from photometric reconstruction techniques (e.g., NeRF, Gaussian Splatting) by employing alpha-transparent training while explicitly distinguishing foreground occlusions from background subtraction. We validate the full pipeline through extensive experiments, demonstrating its effectiveness across diverse objects. By eliminating the need for manual intervention or environment modifications, our pipeline can be integrated directly into existing pick-and-place setups, enabling scalable and efficient dataset creation.

System Overview

Our pipeline integrates seamlessly into a standard pick-and-place setup with two bins and a robotic arm. The robot picks an object from the first bin, interacts with it to gather data for asset creation, and places it in the second bin. This process runs fully autonomously until the first bin is empty. The robot begins by moving the object in front of a static RGBD camera, ensuring full observability through re-grasps as needed. Background pixels are masked using video segmentation, and object tracking determines the object poses in the camera frame. These poses, along with image data, are input to a 3D reconstruction method. We propose a general recipe for obtaining object-centric triangle meshes from arbitrary photometric reconstruction methods, such as NeRF or Gaussian Splatting, by adapting them to dynamic, object-centric scenarios. The collision geometry is then derived from the visual geometry via convex decomposition, simplifying and convexifying the shape for simulation purposes. Lastly, the robot executes an optimal excitation trajectory to collect joint position and torque data, enabling identification of the object's physical properties through convex optimization.

Recording Playback Instructions

We recommend interacting with the simulation recordings:

First, click the Open Controls button at the top right corner to open the recording controls. Second, click the now visible Scene button to collapse the unnecessary options. Third, click the play button to play the recording from the start. The playback speed can be adjusted using the timeScale input. It is possible to move around the simulation recording using mouse controls. If you see "No connection", wait a few seconds for the simulation to finish loading.
